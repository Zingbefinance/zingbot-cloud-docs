# 📘 ZingBot Cloud - Decisions Log

Ce document enregistre toutes les décisions importantes du projet.

---

## Décision #001

Date : 29 juin 2026

Le MVP de ZingBot Cloud sera volontairement simple.

Objectif :

Permettre aux projets Web3 de gérer leur communauté Telegram.

---

## Décision #002

Le développement suivra cette règle :

Une fonctionnalité terminée avant d'en commencer une autre.

---

## Décision #003

Architecture officielle :

- Backend : FastAPI
- Frontend : Next.js
- Base de données : PostgreSQL
- Authentification : JWT
- Cache : Redis
- Déploiement : Docker

---

## Décision #004

Les fonctionnalités avancées sont reportées après le MVP :

- AI Assistant
- Marketplace
- Discord
- X (Twitter)
- Token Center avancé
- Trading Automation

---

## Décision #005

La qualité est prioritaire sur la quantité.

Nous préférons un petit produit stable qu'une grande plateforme inachevée.
